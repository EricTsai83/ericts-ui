# Product

## Register

product

## Users

Developers evaluating and installing ericts/ui components, hooks, and blocks through a shadcn-compatible registry.

## Product Purpose

This site documents the component registry, shows available components, hooks, and blocks, and gives developers a predictable path from discovery to installation or preview.

## Brand Personality

Precise, practical, restrained. The interface should feel like a trustworthy developer tool rather than a marketing showcase.

## Anti-references

Avoid decorative landing-page patterns, oversized hero treatments, noisy gradients, and invented navigation affordances that slow down documentation browsing.

## Design Principles

Use familiar documentation patterns.
Keep navigation compact and scannable.
Prioritize registry utility over decoration.
Make interactive controls predictable and accessible.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard-accessible navigation. Preserve readable focus states, semantic links, and clear labels for icon-only controls.
